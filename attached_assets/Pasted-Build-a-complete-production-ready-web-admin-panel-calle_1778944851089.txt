Build a complete production-ready web admin panel called GaamOps 
for managing the GaamRide mobile app.

GaamRide is a village ride + logistics platform in rural Gujarat, India.
Two modules: GaamRide (person transport) + GaamHaul (tempo/truck for farmers)
Service area: 9 villages in Mahuva taluka, Surat district, Gujarat.

---

## TECH STACK

- React.js (with Vite)
- Firebase Firestore (same database as Flutter app)
- Firebase Auth (admin Google login only)
- Tailwind CSS (styling)
- Recharts (charts and graphs)
- React Router (navigation)
- Google Maps JS API (live tracking map)

---

## FIREBASE COLLECTIONS (same as Flutter app)

users/{userId}          → all registered users
saathis/{userId}        → ride drivers  
rides/{rideId}          → ride bookings
haul_vehicles/{userId}  → tempo/truck owners
haul_bookings/{id}      → haul bookings
villages/{villageId}    → 9 approved villages

---

## WHO CAN ACCESS GAAMOPS

Only specific admin emails (hardcoded whitelist):
- Check Firebase Auth email against allowed list
- If not admin → show "Access Denied" page
- No self-registration, admin-only Google login

## REAL-TIME FEATURES

Use Firestore onSnapshot for live updates on:
1. Dashboard stats (rides count, users count)
2. Live rides map (Saathi positions)
3. Activity feed
4. Active ride/haul status cards

Do NOT poll. Use listeners everywhere.
Unsubscribe listeners on component unmount.

## IMPORTANT REQUIREMENTS

1. Same Firebase project as Flutter app — reads same Firestore data
2. Admin auth: Google login only, whitelist email check
3. All Firestore queries optimized — use indexes, limit results
4. Loading skeletons on every data fetch (not spinners)
5. Empty states with helpful messages
6. Confirm dialogs before destructive actions (block, delete)
7. Toast notifications for all admin actions
8. All monetary values formatted as ₹X,XXX
9. All timestamps in IST (India Standard Time)
10. Gujarati village names shown alongside English

Build complete working code for all pages and components.
Connect to Firebase using environment variables for config.
