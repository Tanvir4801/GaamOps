## BUG 1 — Map marker "BOTTOM OVERFLOWED BY 5.0 PIXELS"

In custom marker builder, the widget overflows.
Fix by adding overflow: Overflow.visible or reducing size:

```dart
// In marker_helper.dart or wherever custom markers are built:

static Future<BitmapDescriptor> buildSaathiMarker(
    String name) async {
  // Use a fixed size that won't overflow
  final recorder = ui.PictureRecorder();
  final canvas = Canvas(recorder);
  const size = Size(80, 90); // fixed safe size

  // Draw circle background
  final paint = Paint()..color = const Color(0xFF2E7D32);
  canvas.drawCircle(
    Offset(size.width / 2, size.height / 2 - 8),
    28, paint);

  // Draw icon (avoid text to prevent overflow)
  // Use only the circle + icon, NO text label below marker
  // Text label causes the overflow

  final picture = recorder.endRecording();
  final image = await picture.toImage(
    size.width.toInt(), size.height.toInt());
  final bytes = await image.toByteData(
    format: ui.ImageByteFormat.png);

  return BitmapDescriptor.fromBytes(bytes!.buffer.asUint8List());
}
```

Future<void> _checkVersion() async {
  try {
    const currentVersion = '1.0.0';

class ConnectivityBanner extends StatefulWidget {}

class _ConnectivityBannerState extends State {
  bool _isOnline = true;

  @override
  void initState() {
    super.initState();
    Connectivity().onConnectivityChanged.listen((results) {
      setState(() {
        _isOnline = results.any((r) =>
          r != ConnectivityResult.none);
      });
    });
  }

  @override
  Widget build(BuildContext context) {
    if (_isOnline) return SizedBox.shrink();
    return Container(
      width: double.infinity,
